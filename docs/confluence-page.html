Key Features

⚡

Multi-Step Execution

Execute multiple APIs sequentially. Auto-bind previous response values to next request parameters.

🔗

Template Variables

{{step.1.data.id}}, {{env.TOKEN}}, {{$randomInt}} and more dynamic variables.

🧭

Multi-Spec Detection

Detects multiple Swagger specs on a single page and merges all endpoints for selection.

📥

History → Shortcut

Convert execution history to reusable shortcuts. Auto-infers path params, step output bindings.

✏️

Endpoint Editing

Select endpoint from spec dropdown, then freely edit method/path for custom requests.

📋

Step Copy / Paste

Copy/paste steps within a shortcut. step.N references auto-renumber on move/remove/paste.

Installation

  1. Clone the repo and run npm install && npm run build
  2. Open Chrome and go to chrome://extensions
  3. Enable Developer mode (top right toggle)
  4. Click Load unpacked
  5. Select the output/chrome-mv3 folder
  6. Navigate to any Swagger UI page — the side panel activates automatically ✅

Template Variable Types

1. Environment Variables — {{env.VAR_NAME}}

References values set in the Env tab.

{{env.BASE_URL}}/api/v1/users
Header: Authorization: Bearer {{env.TOKEN}}

2. Step References — {{step.N.path}}

References extracted values from a previous step's response.

{{step.1.data.id}} — data.id from step 1
{{step.2.orderId}} — extracted orderId from step 2
{{step.1.data.items.0.name}} — first array element

3. Bare Variables — {{varName}}

Searches extracted values by name across previous steps (most recent first), then env variables.

URL: /orders/{{orderId}}/accept
Body: {"amount": {{amount}}}

4. Random Generators — {{$generator}}

{{$randomInt(1,100)}}Random integer in range
{{$randomString(8)}}Random N-char string
{{$randomEmail}}Random email address
{{$uuid}}UUID v4
{{$timestamp}}Current Unix timestamp

5. Array Filter — array[?condition].field

Extract values from arrays using filter conditions in extractor paths.

items[?status==ACTIVE].id
users[?role==admin].email
orders[?type==EXPRESS].trackingId

Supports nested field access and template variable combinations.

Usage Examples

Example 1: Order Fulfillment Flow

Step 1 POST Create Order → Sleep 1s wait → Step 3 GET Order (retry 5x) → Step 4 Accept Order → Step 5 Start Pickup → Step 6 Complete Pickup → Step 7 Complete Delivery
Key: Create an order → wait for processing → query by ID with retry → extract orderId and stepIds → chain through the fulfillment flow using extracted values.

Example 2: History → Edit → Reuse

History tab: Select execution → Convert to Shortcut

Step 1: GET /orders/{orderId}
  Select endpoint from spec dropdown

Step 2: Edit path directly
  /orders/{orderId}/steps/{stepId}/start

Step 3: Copy/paste step, adjust references
  step.2.stepId1 → step.2.stepId2

You can edit method/path after selecting from the spec, and step references auto-renumber on paste.

Example 3: Environment Variables

Env settings:
  BASE_URL = https://api.staging.example.com
  USER_ID = user_abc123
  TOKEN = eyJhbGci...

Step 1: PUT {{env.BASE_URL}}/orders/{{orderId}}/accept
  Header: Authorization: Bearer {{env.TOKEN}}
  Header: x-user-id: {{env.USER_ID}}

Bare variables like {{USER_ID}} are also auto-resolved from env.
