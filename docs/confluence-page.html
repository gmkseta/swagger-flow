템플릿 변수 종류

1. 환경변수 — {{env.변수명}}

Env 탭에서 설정한 값을 참조합니다.

{{env.BASE_URL}}/api/v1/users
Header: Authorization: Bearer {{env.TOKEN}}
Header: x-flexer-accountid: {{env.FLEXER_ID}}

2. 스텝 참조 — {{step.N.경로}}

이전 스텝의 응답에서 추출한 값을 참조합니다.

{{step.1.data.id}} → 스텝1 응답의 data.id
{{step.2.orderId}} → 스텝2에서 추출한 orderId
{{step.1.data.items.0.name}} → 배열 첫번째 요소

3. Bare 변수 — {{변수명}}

이전 스텝에서 추출한 값을 이름만으로 참조. 가장 최근 스텝부터 역순 탐색합니다.

URL: /orders/{{orderId}}/accept
Body: {"income_amount": {{income_amount}}}
Header: x-flexer-accountid: {{FLEXER_ID}}

💡 env 변수와 이전 스텝 추출값 모두 자동 탐색됩니다.

4. 랜덤 생성기 — {{$generator}}

{{$randomInt(1,100)}}범위 내 랜덤 정수
{{$randomString(8)}}N자리 랜덤 문자열
{{$randomEmail}}랜덤 이메일
{{$uuid}}UUID v4
{{$timestamp}}현재 Unix 타임스탬프

5. 배열 필터 추출 — array[?조건].필드

Extractor path에서 배열 내 조건 매칭으로 값 추출.

orders[?vendor.vendor_order_id=={{step.1.doboOrderId}}].open_order_id
orders[?status==ACTIVE].id
items[?type==FOOD].name

💡 중첩 필드 접근(vendor.vendor_order_id)과 템플릿 변수 조합 가능

활용 예시

예시 1: 바로고 통합오더 도보기사 수행 (퀵 host 조합)

Step 1 POST 오더 생성 → Sleep 1초 대기 → Step 3 오더 조회 (retry 5회) → Step 4 오더 수락 → Step 5 픽업 스타트 → Step 6 픽업 완료 → Step 7 배송 완료
핵심: GOA에서 통합오더 생성 → quick 공유 대기 → vendor_order_id 기준으로 오더 조회 → orderId, income_amount, stepId1, stepId2를 추출해서 quick host의 DOBO order-service 플로우로 이어서 수행

예시 2: 히스토리 Import 후 endpoint 직접 수정

History 탭: Execute 결과 선택 → Convert to Shortcut

Step 1: GET [Internal] /orders/{orderId}
  드롭다운으로 endpoint 선택

Step 2: Path 직접 수정
  /orders/{orderId}/steps/{stepId1}/start

Step 3: Copy / Paste로 finish step 복제
  step.2.stepId1 → step.2.stepId2 로 수정

💡 spec endpoint를 고른 뒤에도 method/path를 직접 수정할 수 있고, step 복붙 시 참조도 같이 재정렬됩니다.

예시 3: Env 변수 활용

Env 설정:
  BASE_URL = http://alpha.api.example.com
  FLEXER_ID = FL_ABC123
  TOKEN = eyJhbGci...

Step 1: PUT {{env.BASE_URL}}/orders/{{orderId}}/accept
  Header: Authorization: Bearer {{env.TOKEN}}
  Header: x-flexer-accountid: {{env.FLEXER_ID}}

💡 Bare 변수 {{FLEXER_ID}}도 env에서 자동 탐색됩니다.
